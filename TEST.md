# TEST.md — запуск всех видов тестов в репозитории Re:Earth Visualizer

Документ перечисляет **каждую команду**, **рабочий каталог**, **переменные окружения**, **внешние зависимости** (Mongo, Docker, браузер для Playwright) и **ожидаемый результат**. Ничего из ниже перечисленного не является «опциональным размытым советом»: если шаг помечен как обязательный для вида тестов, без него этот вид тестов **не запускается корректно**.

- **Корень репозитория** — каталог, где лежат `server/`, `web/`, `e2e/`.
- **Успех** — процесс завершился с **кодом выхода 0**.

## 1. Тесты сервера (Go, каталог `server/`)

### 1.1. Юнит- и интеграционные тесты всех пакетов (основной прогон)

| Поле | Значение |
|------|----------|
| Рабочий каталог | `server/` |
| Команда (через Makefile, рекомендуется) | `make test` |
| Эквивалент без Makefile | `go test ./...` |
| Переменные окружения | Для тестов, которым нужна MongoDB на localhost, задайте **`REEARTH_DB=mongodb://localhost`** (как в [`.github/workflows/ci_server.yml`](.github/workflows/ci_server.yml)). Если Mongo **не** запущена, часть тестов может **падать** или **пропускаться** — это не «зелёный» полный прогон. |
| Внешние сервисы | **MongoDB** на `localhost:27017` для полного прогона в стиле CI. Проще всего: поднять стенд `make d-run` из `server/` **до** запуска тестов **или** отдельно контейнер `mongo:8` на порт 27017. |
| Ожидаемый результат | Вывод `ok` по пакетам, код выхода **0**. |

**Команда одной строкой (из корня репозитория):**

```bash
cd server && REEARTH_DB=mongodb://localhost make test
```

### 3.2. API-тесты Playwright (включая сценарий двух WebSocket-клиентов collab)

| Поле | Значение |
|------|----------|
| Рабочий каталог | `e2e/` |
| Команда для **локального** стенда с mock auth | `npm run test:api:local` |
| Что задаёт скрипт (см. [e2e/package.json](e2e/package.json)) | `SKIP_STORAGE_STATE=true`, `REEARTH_E2E_AUTH_MODE=mock`, `REEARTH_WEB_E2E_BASEURL=http://localhost:3000`, затем `npx playwright test --config=playwright.config.ts --project=api-setup --project=api-tests` |
| Предусловия | **Должны быть запущены** и доступны: фронт **`http://localhost:3000`** и бэкенд, с которым этот фронт связан (обычно **`http://localhost:8080`**). Иначе тесты **упадут по таймауту** при сетевых запросах. |

**Ожидаемый результат:** Playwright завершил проекты `api-setup` и `api-tests` без падений, код **0**.

## 4. Рекомендуемый порядок полного локального «как на защите»

Выполняйте **последовательно**. Каждый шаг должен завершиться кодом **0** перед следующим.

1. Поднять Mongo (например через `cd server && make d-run`) и при необходимости fake GCS, если ваши тесты это требуют.  
2. `cd server && REEARTH_DB=mongodb://localhost make test`  
3. `cd web && yarn install && yarn type && yarn lint && yarn coverage && yarn i18n --fail-on-update`  
4. Поднять фронт и бэкенд для e2e (см. [START.md](START.md)).  
5. `cd e2e && npm ci && npm run test:api:local`

## 5. Что этот документ намеренно не описывает

- Запуск **внутренних** скриптов Husky / pre-commit — это не отдельный «вид тестов проекта», а хуки git.  
- Публикация отчётов Allure в облако — шаг CI в [playwright_api_tests.yml](.github/workflows/playwright_api_tests.yml), **не обязателен** для локальной защиты.  
- Нагрузочное и security-тестирование вне репозитория.

## 6. Быстрая шпаргалка команд

```bash
# Сервер (нужен Mongo на localhost для полного прогона)
cd server && REEARTH_DB=mongodb://localhost make test

# Сервер e2e (нужны сервисы — см. server/README)
cd server && make e2e

# Web: типы, линт, покрытие, i18n (однопроходный Vitest: yarn test -- run)
cd web && yarn install && yarn type && yarn lint && yarn coverage && yarn i18n --fail-on-update

# Playwright API локально (нужны web:3000 и API)
cd e2e && npm ci && npm run test:api:local
```

Для запуска приложения перед пунктом «Playwright API локально» следуйте [START.md](START.md).
